# Training Plan — Enterprise Knowledge Platform

A step-by-step improvement plan for the [Enterprise Knowledge Intelligence Platform](https://github.com/RahilMirchiwala/Enterprise-Knowledge-Platform), structured to build professional data science skills and prepare for a role at [Jeavio](https://jeavio.com).

---

## The Big Picture

You built something real. Six experiments, a working API, a layered architecture. That's more than most CS students ever ship. This plan does not ask you to start over — it asks you to finish what you started, measure it properly, and deploy it so the world can see it.

Each phase addresses a gap between where the project is and where production AI systems need to be. Every task has a reason. Read the reasons — they matter more than the tasks.

---

## Phases

| Phase | Focus | Timeline | What You'll Learn |
|---|---|---|---|
| [Phase 1](./phase-1-fix-and-professionalize.md) | Fix what's broken — consistency, empty files, stale docs, error handling | 1–2 weeks | Closing the loop. Professional discipline. |
| [Phase 2](./phase-2-measure-like-a-professional.md) | Proper ML evaluation metrics — Precision, Recall, F1, MRR, MLflow | 2–3 weeks | How to measure what you built, not just whether it worked |
| [Phase 3](./phase-3-replace-toy-search-with-vector-store.md) | Replace TF-IDF with a vector database (ChromaDB + SBERT) | 2–3 weeks | Production ML architecture. Research vs production design. |
| [Phase 4](./phase-4-complete-the-product.md) | Build the duplicate detection feature that was the original goal | 2–3 weeks | Closing a product loop. Business problem → working feature. |
| [Phase 5](./phase-5-deploy-it.md) | Deploy to Render, add health checks, record a demo | 1 week | What deployment actually means and why it changes how you build |
| [Phase 6](./phase-6-interview-preparation.md) | Interview prep — your project, ML fundamentals, system design, Jeavio culture | Ongoing | How to explain what you built and how you think |

---

## One Principle to Carry Through All of It

> **"Why" matters more than "what".**

Every task in this plan has a reason. If you follow the tasks without understanding the reasons, you'll have a better portfolio but you won't have learned much. If you understand the reasons deeply enough to explain them to someone else, you'll be ready to work on systems you haven't seen yet — which is exactly what professional engineering requires.

Read the reasoning sections. Argue with them if you disagree. That thinking is the work.
